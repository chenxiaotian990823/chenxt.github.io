# 数据库基本操作

TIP

  • 【登录】 mysql -u 用户名 -p密码;
  • 【查看所有数据库】 show databse;
  • 【使用某个数据库】 use databaseName;
  • 【查看数据库下的表】 show tables;
  • 【查看某个表】 desc 表名;
  • 【创建数据库】 create database [if not exists] 数据库名字;
  • 【删除数据库】 drop database [if exists] 数据库名字;
  • 【查询数据库特殊情况】 select user from tableName; 以上写法会报错，改写为 select `user` from tableName;

# 数据库的数据(列)类型

TIP

数值

类型名称
tnyint 十分小的数据 1个字节
smallint 较小的数据 2个字节
mediumint 中等大小的数据 3个字节
int(常用) 标准的整数 4个字节
bigint 较大的数据 8个字节

double 浮点数 8个字节

类型名称

varchar(常用,对应java中的String类型) 可变字符串 0~65535
tinytext 微型文本 2^8 - 1
text(保存大文本) 文本串 2^16 - 1

TIP

时间日期 java.util.Date

类型名称

datetime(常用) YYYY-MM-DD HH:MM:SS 最常用的时间格式
timestamp 时间戳 1970.1.1 ~ 至今的毫秒数
year 年

TIP

null
没有值，未知
不要使用NULL进行运算,结果一定是null

# 数据库的字段属性(重点)

Unsigned:

  • 无符号的整数
  • 声明了该列，不能声明为负数

zerofill

  • 0填充
  • 不足的位数用0填充
  • eg: int(3) 5 --> 005

自增

  • 通常理解为自增，自动在上一条记录的基础上+1
  • 通常用来设计唯一的主键，必须是整数类型
  • 可以自定义设计主键自增的起始值和步长

null 和 not null

  • not null非空
  • 假设设置为not null ,如果不给赋值，就会报错
  • null，如果不填写值，默认就是null

默认

  • 比如sex ，默认值是男，如果不指定该列的值，则会有默认的值

(拓展)每一个表，都必须存在以下5个字段！未来做项目用的，表示一个记录存在的意义

  • id 主键
  • `version` 乐观锁
  • is_delete 伪删除
  • gmt_create 创建时间
  • gmt_update 修改时间

# 创建数据库表(重点)

格式

CREATE TABLE [IF NOT EXISTS] `表名`(  
    '字段名' 列类型[属性][索引][注释],
    '字段名' 列类型[属性][索引][注释],
    ......
    '字段名' 列类型[属性][索引][注释],
)[表类型][字符集设置][注释]

偷懒操作

  • SHOW CREATE DATABASE school --查看创建数据库的语句
  • SHOW CREATE TABLE student --查看student数据表的定义语句
  • DESC student --查看student表的结构

# 数据表的类型

关于数据库引擎

INNODB 默认使用

MYISEM 早些年使用的

比较内容 MYISAM INNODB
事务支持 不支持 支持
数据行锁定 不支持 支持
外键约束 不支持 支持
全文索引 支持 不支持
表空间的大小 较小 较大，约为MYISAM的两倍

常规使用操作:

  • MYISAM 节约空间，速度较快
  • INNODB 安全性高，支持事务的处理，多表多用户操作

在物理空间存在的位置

所有的数据库文件都放在data目录下

本质还是文件的存储

MYSQL数据库引擎区别

  • INNODB在数据库表中只有一个*.frm文件，以及上级目录下的ibdata1文件
  • MYISAM对应文件
    1. .frm -表结构的定义文件
    2. .MYD -数据文件(data)
    3. .MYI -索引文件(index)

设置数据库表的字符集编码

CHARSET=utf8

如果不设置，会是mysql默认的字符集编码（默认不支持中文） mysql默认编码是Latin1，不支持中文 在my.ini中配置默认的编码

character-set-server=utf8

# 修改和删除表

修改

  • 修改表名 
    ALTER TABLE student RENAME teacher; 

修改表名公式  
ALTER TABLE 旧表名 RENAME 新表名;
  • 增加表的字段 
    ALTER TABLE student ADD age INT(11);  

增加表字段公式
ALTER TABLE 表名 ADD 字段名 列属性;
  • 修改表的字段
    • 重命名 
      ALTER TABLE student CHANGE age age1 INT(1);
公式
ALTER TABLE 表名 CHANGE 旧名字 新字段名字 列属性;
    • 修改约束 
      ALTER TABLE student MODIFY age VARCHAR(11);
公式
ALTER TABLE 表名 MODIFY 字段名字 列属性;
      • change 用来字段重命名，不能修改字段类型和约束; modify 不用来字段重命名，只能修改字段类型和约束
  • 删除表的字段 
    ALTER TABLE student DROP age;

删除

  • 删除表(如果表存在再删除) 
    DROP TABLE IF EXISTS student;

WARNING

所有的创建和删除操作尽量加上判断条件，以免报错

注意点

  • ``所有的字段名，使用``包裹
  • 注释用 -- 或者/**/
  • sql关键字大小写不敏感，建议写小写
  • 所有的符号用英文
